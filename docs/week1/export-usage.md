# 📊 データエクスポート機能 使い方ガイド

## 🎯 概要

経費データをExcel/PDF形式でエクスポートする機能です。

---

## 🚀 使い方

### 1. 経費一覧のExcelエクスポート

#### **Web画面から**
1. 経費一覧画面（`/expenses/search`）にアクセス
2. 検索条件を指定（任意）
3. 「Excelダウンロード」ボタンをクリック
4. Excel形式（.xlsx）でダウンロード

#### **API呼び出し**
```bash
curl -X GET "http://localhost:8080/api/exports/excel/expenses?status=APPROVED" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -o expenses.xlsx
```

**出力例:**

## 🔧 技術仕様

### Excelファイル

- **ライブラリ**: Apache POI 5.2.5
- **形式**: .xlsx（Excel 2007以降）
- **機能**:
  - ヘッダー行（背景色: 青、フォント: 白・太字）
  - データ行（枠線付き）
  - 通貨フォーマット（¥123,456）
  - 合計行（件数、合計金額）
  - 列幅の自動調整

### PDFファイル

- **ライブラリ**: OpenPDF 1.3.32
- **形式**: PDF
- **機能**:
  - タイトル（中央揃え、太字）
  - 情報テーブル（2カラム）
  - 一覧テーブル（7カラム、ヘッダー背景色付き）
  - フッター（自動生成日時）

## ❓ トラブルシューティング

### エラー: "ファイルが開けません"

**原因:**
- ファイルが破損している
- MIMEタイプが正しくない

**解決策:**
```java
// Content-Typeヘッダーを確認
headers.setContentType(MediaType.parseMediaType("application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"));
```

### エラー: "日本語ファイル名が文字化けする"

**原因:**
- ファイル名がURLエンコードされていない

**解決策:**
```java
String filename = URLEncoder.encode("経費一覧.xlsx", StandardCharsets.UTF_8)
        .replaceAll("\\+", "%20");
```

### パフォーマンス問題: "大量データで遅い"

**原因:**
- 全データをメモリに読み込んでいる

**解決策（将来の改善）:**
- ストリーミング処理の導入
- ページネーションの活用
- 非同期エクスポート
