# 🎓 1週目: テスト完全マスター（初学者向け）

## 📖 このカリキュラムについて

**対象者:** プログラミング初学者・テストを書いたことがない人

**目標:**
- テストの重要性を理解する
- 基本的なテストを自分で書けるようになる
- 既存のテストコードを改善できるようになる

**学習時間:** 1日1-2時間 × 7日間 = 合計10時間程度

---

## 📅 7日間の学習スケジュール

### Day 1: テストとは何か？なぜ必要か？ 🤔

**学習内容:**
- テストがない世界の恐怖
- テストがある世界の安心感
- テストを書くメリット
- テストの種類（単体・統合・E2E）

**成果物:**
- テストの重要性を理解する
- テストピラミッドを理解する

**所要時間:** 30分

---

### Day 2: 超シンプルなテストを書く ✍️

**学習内容:**
- 最もシンプルなテストを書く（計算ロジック）
- JUnit 5の基本
- @Test アノテーションの使い方
- assertEquals, assertTrue の使い方

**成果物:**
- 自分で最初のテストを書く
- テストが通る/落ちるを体験する

**所要時間:** 1.5時間

**演習:**
```java
// ページネーションのロジックをテスト
@Test
void ページ番号リストを正しく生成できる() {
    // ここに実装
}
```

---

### Day 3: AssertJで検証する 🔍

**学習内容:**
- AssertJとは？
- assertThat().isEqualTo() の使い方
- よく使うアサーションメソッド
- エラーメッセージをわかりやすくする

**成果物:**
- AssertJを使ったテストを書ける
- テストが失敗したときのメッセージを理解できる

**所要時間:** 1.5時間

**演習:**
```java
// normalizedOrderBy メソッドをテスト
@Test
void 不正なソート条件はデフォルト値にフォールバック() {
    // ここに実装
}
```

---

### Day 4: HTTPリクエストのテスト 🌐

**学習内容:**
- MockMvcとは？
- GET/POSTリクエストのテスト
- httpBasic（認証）の使い方
- jsonPath（JSONレスポンスの検証）

**成果物:**
- APIのテストを書ける
- レスポンスを検証できる

**所要時間:** 2時間

**演習:**
```java
// 経費提出APIをテスト
@Test
void 下書きの経費を提出できる() {
    // ここに実装
}
```

---

### Day 5: テストの整理とリファクタリング 🧹

**学習内容:**
- @Nested でグループ化
- @DisplayName で日本語のテスト名
- @BeforeEach でテストデータを準備
- Given-When-Thenパターン

**成果物:**
- 読みやすいテストを書ける
- テストを機能ごとにグループ化できる

**所要時間:** 2時間

**演習:**
```java
// 既存のテストをリファクタリング
@Nested
@DisplayName("経費提出API")
class SubmitTest {
    // ここに実装
}
```

---

### Day 6: エラーケースのテスト ⚠️

**学習内容:**
- 例外のテスト方法
- assertThatThrownBy の使い方
- ステータスコードの検証
- エラーメッセージの検証

**成果物:**
- 異常系のテストを書ける
- エラーハンドリングを検証できる

**所要時間:** 1.5時間

**演習:**
```java
// エラーケースをテスト
@Test
void 存在しない経費を提出すると404エラー() {
    // ここに実装
}
```

---

### Day 7: 演習問題と総復習 📝

**学習内容:**
- 1週間の復習
- 総合演習問題
- 既存のテストコードを改善

**成果物:**
- 自分でテストを書けるようになる
- 既存のコードを改善できる

**所要時間:** 2時間

**演習:**
```java
// 経費承認APIの完全なテストを書く
@Nested
@DisplayName("経費承認API")
class ApproveTest {
    // 正常系、異常系、境界値をすべてテスト
}
```

## 🚀 学習の進め方

### 推奨の学習方法

1. **理論を読む** - まずその日の資料を読む（15分）
2. **実例を見る** - サンプルコードを見る（15分）
3. **自分で書く** - 演習問題を解く（30-60分）
4. **答え合わせ** - 解答例と比較する（15分）
5. **復習** - わからなかった部分を見直す（15分）

### つまずいたら

- **わからない用語** → その日の資料に戻る
- **エラーが出た** → エラーメッセージを読む
- **テストが通らない** → Given-When-Thenを確認
- **それでもダメ** → 解答例を見る（恥ずかしくない！）
